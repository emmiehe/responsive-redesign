Identifying Usability Problems

In this part, you will be picking a one page of a website to analyze based on usability and accessibility standards. You will then redesign and build the redesigned version of this webpage. Picking a Website Pick a webpage that is publicly accessible without creating an account. Brown-specific sites like Banner and ASK are NOT publicly accessible. We recommend avoiding mass consumer websites with dedicated UI/UX designer(s), like Facebook or Twitter. Instead, choose an easier interface to improve, like websites for a local hospital, network router support, a custom-made restaurant ordering site, local government service, etc. Feel free to check your chosen interface with a TA. After you choose an interface to redesign, take a screenshot and link to it. Finding Problems Analyze the usability of the website using the criteria we learned about in class: usability, learnability, and memorability. Write about these problems, in the form of a list, table, infographic, or bullet points (depending on how you would like to present it in your portfolio!). It might be helpful to refer back to the Visual Design and Layout lectures. Accessibility In addition to analyzing the website on the four usability guidelines, use WebAIM WAVE to detect any possible accessibility concerns. Use a screen-reader to test how the website is interpreted by screen-readers. Write a few sentences about your findings.

Visual Redesign

Keep in mind that the screen needs to be responsive - that is, a different layout for mobile, tablet, and desktop. We recommend you split up the page into different grids and containers, so that you can easily use flexbox or grids for simple responsive development.

Low-fidelity Wireframing

Using a low-fidelity prototyping software (Balsamiq or Invision), create a total of 3 wireframes of your screen, one for each of the screen sizes (mobile, tablet, desktop). You will be submitting annotated wireframes in your handin. Annotations can simply be labels pointed to where they apply, but just make sure they are visually distinct so they aren't confused for part of the prototype itself. For each wireframe, consider the problems you described in part 1. Annotate where your wireframe solves those problems and describe how.

High-Fidelity Prototyping

Using a high-fidelity prototyping software (Figma or AdobeXD), create a total of 3 high-fidelity prototypes of your screen, one for each of the screen sizes (mobile, tablet, desktop). These should be based on the low-fidelity wireframes you created. For each prototype, annotate the different elements so that an engineer would be able to take your designs and reproduce them without needing additional explanation. Annotate how your website would be responsive to handle the different screen sizes. Is it clear how the styling and/or structuring of each element will change when you move from desktop to mobile to tablet? Here are some example questions to consider annotating: For larger containers, will you be using flexbox, grid, or something else? Why? Does the page change because of a user-action (such as a click or a hover)? How?

Visual Design Style Guide

For any elements that change on events (e.g., on hover, on click), keeping their interactions consistent throughout a product keeps the user experience coherent. Clear documentation also makes handoffs to developers easier. Here is an example. Create a visual design style guide that displays the main colors, typography, and reusable components’ different states (buttons on hover, click, etc.). We expect to see all major components included in your style guide, but how many components this totals to will vary based on your design choices. Focus on interactive elements like input fields over colors and fonts.

Responsive Redesign

Using the final high-fidelity prototypes, create the page using HTML and CSS! Make sure you’ve thoroughly tested for responsiveness. Here are some things to try: Play with the size of your browser to see how the website looks on other computer screens Try changing the font size on your browser Use Developer Tools, e.g. the Chrome Responsive Tool or Firefox Responsive Design Mode to see how the website looks on other devices (optionally) Use Google Translate to see how your website looks on different languages Try WebAIM WAVE on different screen sizes to make sure that your website is still accessible on different devices Write a few sentences or bullet points about your observations.
